SECTION 5.5 -- Evaluation
==============================================================================

  Metrics and Measurement Plan
  - Security Effectiveness (Primary):
      ASR = Successful attacks / Total attacks
      ABR = Blocked attacks / Total attacks
  - Accuracy (Support):
      FP = Legitimate requests incorrectly blocked
      FN = Attack requests not blocked
  - Performance (Critical):
      Average latency (ms), p95 latency (ms), and latency overhead vs baseline
      Latency overhead = Proxy latency - Baseline MCP latency

  Note: Accuracy/precision-only reporting is insufficient for a security
  enforcement proxy; security effectiveness and latency must be included.

  Interpretation (ML Metrics):
  - Precision: Of all requests blocked, what % were actual attacks? (not false alarms)
  - Recall: Of all actual attacks, what % did we successfully catch?
  - F1 Score: Harmonic mean of Precision & Recall (0-100, higher is better)
  - TP/FP/FN/TN: Confusion matrix — raw counts for detailed analysis

  Connection to Security Metrics:
  - High Precision (close to 100%) + High Recall (close to 100%) + High F1 (close to 100%)
    confirms that the proxy is both effective (catches attacks) and precise (no false alarms)
